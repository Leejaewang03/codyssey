### 📄 텍스트 파일 vs 💾 이진(Binary) 파일 비교 분석

| 구분 | 텍스트 파일 (Text File) | 이진 파일 (Binary File) |
| :--- | :--- | :--- |
| **저장 형태** | 사람이 읽을 수 있는 문자(Character) 형태 (예: UTF-8, ASCII) | 컴퓨터가 읽기 쉬운 0과 1의 바이트(Byte) 데이터 형태 |
| **대표 확장자** | `.csv`, `.txt`, `.md`, `.json`, `.py` | `.bin`, `.dat`, `.exe`, `.jpg`, `.pkl` (Pickle) |
| **장점 (Pros)** | - 메모장, VSC 등 일반 에디터로 **쉽게 열고 내용 수정 가능**<br>- 호환성이 높아 다양한 프로그램과 운영체제에서 공유하기 좋음<br>- 사람이 직접 눈으로 보고 에러를 찾기(디버깅) 편함 | - 데이터를 문자열로 변환(파싱)할 필요가 없어 **읽고 쓰는 속도가 매우 빠름**<br>- 데이터를 압축된 형태로 저장하여 **저장 공간(용량)을 효율적으로 사용함**<br>- 파이썬의 리스트나 딕셔너리 같은 복잡한 객체 구조를 메모리 상태 그대로 저장 가능 |
| **단점 (Cons)** | - 컴퓨터가 연산하기 위해 문자를 다시 데이터로 변환하는 과정이 필요해 **처리 속도가 상대적으로 느림**<br>- 이진 파일에 비해 저장 용량을 더 많이 차지함 | - 전용 프로그램이나 코드가 없으면 **사람이 눈으로 내용을 읽고 해독할 수 없음** (일반 에디터로 열면 글자가 깨져 보임)<br>- 플랫폼이나 버전에 따라 호환성 문제가 발생할 수 있음 |
